# Orbit EDU MVP (HTML/CSS/JS)

Рабочий MVP интерактивного образовательного сайта на космическую тематику.

## Что это
Проект реализован как SPA на чистых `HTML/CSS/JavaScript` с hash-роутингом (`#/...`), чтобы всё работало локально без сервера и сборки.

- минималистичный космический UI в чёрно-белой гамме;
- главная без классического header, с кликабельными «планетами»;
- полный пользовательский путь: вход → изучение → тест → награда → профиль;
- данные и прогресс сохраняются в `localStorage`.

## Запуск
1. Откройте папку проекта.
2. Откройте файл `index.html` в браузере.
3. Начните с регистрации: `#/auth/register`.

Дополнительно: можно просто открыть `index.html`, роутер сам отправит на `#/auth/login`.

## Реализованные маршруты
В локальном режиме используется формат `#/...`.

- `/`
- `/auth/login`
- `/auth/register`
- `/profile`
- `/profile/history`
- `/profile/projects`
- `/profile/career`
- `/profile/achievements`
- `/sections/[slug]`
- `/sections/[slug]/quiz`
- `/forum`
- `/forum/[id]`

Примеры:
- `#/sections/history`
- `#/sections/modern/quiz`
- `#/forum/topic-1`

## Core flow (MVP сценарий)
1. Пользователь открывает сайт.
2. Регистрируется или входит.
3. На главной выбирает раздел через планету.
4. Изучает материалы (отмечает их как прочитанные).
5. Проходит тест раздела.
6. Получает баллы и награды.
7. Видит результат, историю попыток, награды и достижения в профиле.

## Данные и сущности
Сущности реализованы в `state` и mock-данных:

- `User`
- `Section`
- `MaterialBlock`
- `Quiz`
- `QuizQuestion`
- `QuizAttempt`
- `UserProgress`
- `Achievement`
- `Reward`
- `UserReward`
- `ForumTopic`
- `ForumComment`
- `ActivityEvent`

Источник seed/mock data: `data/content-data.js`.

## Структура
- `index.html` — корневой контейнер приложения и шаблоны компонентов;
- `style.css` — единый монохромный стиль, адаптивность;
- `script.js` — роутер, UI-рендер, бизнес-логика, localStorage;
- `data/content-data.js` — mock-контент разделов, тестов, форума, наград.

## Что упрощено в пользу рабочего MVP
- авторизация локальная (без backend, JWT и шифрования);
- форум упрощён (темы + комментарии, без модерации и вложенности);
- файл достижений сохраняется как метаданные (имя/дата), без физической загрузки на сервер.
